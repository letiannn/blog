# ISO 15031 (第1-6部分)

## **ISO 15031 系列标准文件简介 (Part 1-6)**

本系列标准共同构成了 **“车载诊断系统”** 的国际通用技术规范，是全球范围内实施OBD-II及相关排放法规的基石。它为车辆与外部诊断设备之间的通信，提供了从物理接口到数据内容的完整标准化方案。

### **ISO 15031-1: 一般信息**
*   **核心定位：整套标准的“总纲”和“使用指南”。**
*   **主要内容：**
    1.  **整体介绍：** 阐述了ISO 15031系列标准的**范围、目的及其与OBD法规（如美国OBD-II、欧洲EOBD）的关联**。说明了它如何帮助车辆满足法规的排放监测要求。
    2.  **结构说明：** 概述了系列内**各个部分（Part 2至Part 7）的主要内容和相互关系**，为使用者提供清晰的导航。
    3.  **规范性引用：** 列出了本系列标准所依赖的其他基础国际标准。
    4.  **术语与定义：** 提供了贯穿本系列的核心术语解释。
*   **阅读建议：** 了解OBD诊断标准体系的入门必读部分，是理解后续技术细节的基础。

### **ISO 15031-2: 术语、定义、缩写和首字母缩略语**
*   **核心定位：标准的“专用词典”**
*   **主要内容：**
    1.  **集中定义：** 系统性地列出并解释了在OBD诊断领域，特别是ISO 15031语境下的所有**专业术语、定义、缩写和缩略语**。
    2.  **统一理解：** 确保制造商、设备供应商、维修人员和法规机构对如“MIL”（故障指示灯）、“DTC”（诊断故障码）、“PID”（参数标识）等关键概念有完全一致的理解。
*   **阅读建议：** 作为工具手册，供查阅和澄清概念使用。

### **ISO 15031-3: 诊断连接器及相关电路 — 技术要求**
*   **核心定位：物理层的“统一插头”规范。**
*   **主要内容：**
    1.  **连接器规格：** 强制规定了车辆上**16针OBD诊断连接器（J1962）的物理形状、尺寸、引脚排列**。
    2.  **电气特性：** 详细定义了**每个引脚的功能**（如电源、接地、CAN总线高/低、K线等）、电压电平、通信协议分配及电气负载要求。
    3.  **安装位置：** 建议了连接器在车辆驾驶舱内的**大致安装位置和可达性要求**。
*   **阅读建议：** **硬件工程师和诊断设备制造商的关键文件。** 它确保了所有符合标准的车辆和工具在物理层可以互联。

### **ISO 15031-4: 外部测试设备**
*   **核心定位：诊断设备与车辆的“会话协议”框架。**
*   **主要内容：**
    1.  **通信框架：** 描述了**外部诊断设备**与车辆OBD系统之间进行**诊断通信的通用框架、模式和流程**。
    2.  **功能要求：** 规定了诊断设备为实现法规要求的测试（如I/M检查）所应具备的**基本功能和行为**。
    3.  **交互规范：** 定义了通信的初始化、消息交换、错误处理等高层协议交互过程。
*   **阅读建议：** 规定了诊断工具“如何发起并维持一场有效的对话”，是系统架构师和诊断应用开发者的重要参考。

### **ISO 15031-5: 排放相关诊断服务**
*   **核心定位：诊断数据的“通用词典”和“请求手册”。**
*   **主要内容：**
    1.  **参数标识（PID）定义：** 详细定义了用于请求和获取车辆实时数据的**标准化PID列表及其格式**（如发动机转速-PID 0x0C，车速-PID 0x0D）。
    2.  **模式与服务：** 基于SAE J1979等，规定了获取数据的**诊断服务模式**（如模式$01-请求实时数据，模式$02-请求冻结帧数据）。
    3.  **数据公式与单位：** 为每个PID提供了将原始字节值转换为**具有物理意义和单位（如RPM, km/h, °C）的工程值**的转换公式。
*   **阅读建议：** **诊断软件开发者的核心文件。** 它告诉工具“具体问什么问题”以及“如何解读得到的答案”，是实现数据流读取功能的基础。

### **ISO 15031-6: 诊断故障代码定义**
*   **核心定位：故障信息的“标准疾病代码本”。**
*   **主要内容：**
    1.  **DTC格式规范：** 严格规定了**诊断故障码的标准化结构、编码规则和分组**。例如，P0xxx（动力总成），C0xxx（底盘），B0xxx（车身），U0xxx（网络通信）。
    2.  **标准DTC列表：** 提供了一份**全球统一的、预先定义好的DTC列表**及其对应的故障描述。这是所有车辆制造商都必须遵循的基础代码集（例如P0300-随机/多缸失火检测）。
    3.  **故障类型指示：** 定义了如何通过DTC和状态位来指示故障的类型、发生频率和确认状态。
*   **阅读建议：** **维修技师、法规检测员和诊断信息提供商的必读文件。** 它确保了全球对同一故障现象使用统一代码进行报告和理解。

## **总结关系链**
*   **Part 1 & 2** 告诉您 **“OBD诊断标准是什么”** 和 **“用到的词是什么意思”**。
*   **Part 3** 规定了 **“物理上如何连接”**（统一的插座）。
*   **Part 4** 建立了 **“对话的基本规则和流程”**。
*   **Part 5** 提供了 **“可以问哪些具体问题（PID）以及如何理解答案”**。
*   **Part 6** 定义了 **“如何报告疾病（DTC）及其标准名称”**。

这套标准共同作用，使得任何一辆符合法规的车辆，都能被任何一台符合标准的通用诊断设备进行基本的排放相关诊断，是全球汽车后市场和排放监管体系得以高效运转的技术支柱。
